Database access

The database was designed to accomodate multiple users and collaborators and to maintain its usefulness as its contents grow. Users familiar with Structured Query Language (SQL) can query the database directly from their analytical software of choice via appropriate connection settings (e.g. Open Database Connectivity (ODBC) connection and other examples can be found in the Supporting Information). Database views assist this level of entry by formatting data to be returned in column format such as those typically held in spreadsheets. This entry approach minimizes the risk posed by the alternative static copy, whereby changes enter and are inherited in the process of dissemination (Barbrook et al., 1998, for a literary example).
